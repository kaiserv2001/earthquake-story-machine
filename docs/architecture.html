EARTHQUAKE STORY MACHINE

Event-driven .NET 8 Azure pipeline — USGS quakes → Service Bus → enrichment → story cards → Static Web App

Pipeline

  • • UsgsPollerFunction — timer poll of the USGS GeoJSON feed, de-dupes new quakes
  • • quake-events — Service Bus queue decouples poll from build
  • • StoryBuilderFunction — enriches each quake via 5 external APIs, null-safe per section

Data & Serving

  • • Blob Storage — full story-card JSON per quake
  • • Azure SQL — StoryCards list index (EF Core)
  • • HTTP API — /api/cards, /api/cards/{id}, /api/feed (Atom)

Tech Stack

  • • .NET 8 isolated-worker Azure Functions, single app
  • • Frontend — vanilla JS Static Web App, Leaflet map view
  • • Local dev — Azurite, SQL Server, SB emulator + AMQP readiness gate (docker compose)
